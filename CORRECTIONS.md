## How to Use This File

### Feedback Classification (Signal vs. Noise)

Every correction Dale gives falls into one of three types:

| Tag | Meaning | Action |
|-----|---------|--------|
| `[CARRIER]` | Permanent signal — recurring pattern that should change how I write | Promote to training file after 2-3 confirmed recurrences |
| `[MODULATION]` | Task-local — Dale adjusted something for this specific piece only | Do NOT propagate; discard after task |
| `[CALIBRATION]` | Updates a writing default — e.g. hook style, post length, CTA format | Update default after 2-3 recurrences |

### Category Tags

| Tag | Covers |
|-----|--------|
| `[METHOD]` | Wrong writing technique or approach |
| `[FRAMEWORK]` | Wrong structural framework applied (e.g. wrong Anett Grant pattern) |
| `[FORMAT]` | Output formatting, length, or presentation |
| `[SCOPE]` | Too broad, too narrow, or wrong topical scope |
| `[INTERP]` | Misread Dale's intent for the piece |
| `[TONE]` | Voice too formal/casual, wrong register for LinkedIn |
| `[STRUCTURE]` | Hook/body/CTA organization issues |

### Promotion Rule

Only promote a correction to a training file after 2-3 independent recurrences of the same category + type.
Single occurrences go in Pending. Noise Archive for true one-offs.
